PMM 3.7.0 introduces Real-time Analytics (RTA) API endpoints for programmatic access to live query execution data on your MongoDB services.

## New features

### Real-time Analytics API (Technical Preview - MongoDB only)

While Query Analytics (QAN) helps you optimize performance by analyzing completed queries, the new RTA API shows you what's happening on your databases right now. When your database is struggling, you can use these endpoints to identify problematic operations immediately and take action before they impact users.

#### New endpoints

- **GET /v1/realtime/query-data**: returns currently executing queries with full operation details including query text, duration, execution plan, client information, and lock status
- **POST /v1/realtime/change**: enables or disables real-time monitoring for specific services or entire clusters

#### Common use cases

- **Incident response**: identify problematic queries during active performance issues
- **Automated monitoring**: integrate real-time query data into custom dashboards and alerting systems
- **Cluster-wide visibility**: monitor all services in a MongoDB cluster simultaneously

#### Requirements and limitations

**Requirements:**
- PMM Server 3.7.0 or later
- PMM Client 3.7.0 or later
- MongoDB service monitored by PMM

**Limitations:**
- **Technical Preview**: not recommended for production use
- **MongoDB only**: MySQL and PostgreSQL support planned for future releases
- **Temporary data**: query data reflects current execution only and is not persisted
- **Performance impact**: real-time monitoring adds overhead to MongoDB and PMM
