### HA status API endpoints (Tech Preview)

PMM 3.6.0 introduces High Availability monitoring capabilities, and with this, new API endpoints for programmatic access to HA cluster status. These endpoints support the broader HA monitoring features added in this release, including the HA status indicator, Inventory visibility, and HA Overview dashboard.

For full details on HA monitoring capabilities, see the [PMM 3.6.0 release notes](https://per.co.na/pmm/3.6.0).

### New endpoints

- **GET /v1/ha/nodes** — returns HA cluster node information
- **GET /v1/ha/status** — returns HA mode status

### GET /v1/ha/nodes

Returns information about all nodes in the PMM HA cluster.

**Response fields:**

| Field | Type | Description |
|-------|------|-------------|
| `node_name` | string | Node identifier (e.g., `pmm-ha-0`, `pmm-ha-1`) |
| `role` | string | `NODE_ROLE_LEADER` or `NODE_ROLE_FOLLOWER` |
| `status` | string | Node availability (`alive` or `unavailable`) |

### Example request
```sh
curl -ks "https://admin:password@127.0.0.1:8443/v1/ha/nodes"
```

### Example response
```json
{
  "nodes": [
    {
      "node_name": "pmm-ha-0",
      "role": "NODE_ROLE_LEADER",
      "status": "alive"
    },
    {
      "node_name": "pmm-ha-1",
      "role": "NODE_ROLE_FOLLOWER",
      "status": "alive"
    },
    {
      "node_name": "pmm-ha-2",
      "role": "NODE_ROLE_FOLLOWER",
      "status": "alive"
    }
  ]
}
```

### GET /v1/ha/status

Returns whether PMM is running in HA mode.

### Example request
```sh
curl -ks "https://admin:password@127.0.0.1:8443/v1/ha/status"
```

### Example response
```json
{
  "status": "Enabled"
}
```

**Response values:**

- `Enabled` — PMM is running in HA mode
- `Disabled` — PMM is running in standalone mode

### Cluster health algorithm

The overall cluster health is determined by node availability:

| Status | Condition |
|--------|-----------|
| Healthy | All nodes alive |
| Degraded | 1/3 of nodes unavailable |
| Critical | 2/3 of nodes unavailable |
| Down | All nodes unavailable |

### Impact

These endpoints are available only when you deploy PMM in HA mode on Kubernetes using the Helm chart. If you're running a standalone deployment, `/v1/ha/status` returns `Disabled`. Node names match your Kubernetes pod names. No changes are required for existing integrations.
