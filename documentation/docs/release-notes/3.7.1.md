# Percona Monitoring and Management 3.7.1

**Release date**: May 1st, 2026

Percona Monitoring and Management (PMM) is an open source database monitoring, management, and observability solution for MySQL, PostgreSQL, MongoDB, Valkey and Redis. PMM empowers you to: 

- monitor the health and performance of your database systems
- identify patterns and trends in database behavior
- diagnose and resolve issues faster with actionable insights
- manage databases across on-premises, cloud, and hybrid environments

## ✨ Release highlights

### Spot MongoDB storage fragmentation at a glance

The [MongoDB Cluster Summary](../reference/dashboards/dashboard-mongodb-cluster-summary.md) and [MongoDB ReplSet Summary](../reference/dashboards/dashboard-mongodb-replset-summary.md) dashboards now include a **Fragmentation Analysis** panel showing the ratio of free to allocated storage per collection. Use it to quickly identify collections wasting disk space due to frequent deletes or document moves, and decide where running `compact` will have the most impact.

## 📈 Improvements

- [PMM-14875](https://perconadev.atlassian.net/browse/PMM-14875): Added a **Fragmentation Analysis** panel to the MongoDB Cluster Summary and MongoDB ReplSet Summary dashboards. 

- [PMM-14832](https://perconadev.atlassian.net/browse/PMM-14832): PMM Client now supports a custom path to the `/proc/mounts` file used by the `node_exporter` filesystem collector. Use the new `--proc-mounts-path` flag with `pmm-agent setup` (or the `PMM_AGENT_SETUP_PROC_MOUNTS_PATH` environment variable) to specify the path. This is useful in containerized or non-standard environments where `/proc/mounts` is not at its default location.

## ✅ Fixed issues

- [PMM-14852](https://perconadev.atlassian.net/browse/PMM-14852): Fixed the [MongoDB InMemory Details](../reference/dashboards/dashboard-mongodb-inmemory-details.md) dashboard, where panels for transactions, cache capacity, sessions, and pages displayed no data. The panels were querying WiredTiger-specific metrics instead of InMemory-specific ones. The dashboard also had duplicate and irrelevant template variables that have been removed. All affected panels now display data correctly for MongoDB instances using the InMemory storage engine.
